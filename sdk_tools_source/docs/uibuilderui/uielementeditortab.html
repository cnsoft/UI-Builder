Element Editor Tab

This is the main editor for elements. On the left side is a tree view of all elements, on the right is a list of properties on the currently selected element.

Left clicking on an element in the tree view selects that element as the current element. Right clicking brings up a menu with 'remove' in it, this removes the current element from its parent and from its parents layout. Elements can be dragged into new positions in the tree. To make an element the child of another drop it onto the element, to insert it between two elements drop it into the gap between to them. To create a copy of an element hold <ctrl> when copying. To add a different kind of element, drag an element from the Palette.

The properties of the elements differ depending of the type of the element. See the documentation for details of individual properties. A couple of important properties are:

name

This identifies the element, this needs to be unique compared to the sibling elements, preferably completely unique.

style

This points to a property set specifying default values of this elements parameters. There is a Go To button next to the property, this takes you to the Property Set in the style editor If a parameter is specified in the property set pointed to by style, or one of its parents, then it it treated specially. If the property has been set in this element then there is a Reset button next to it, clicking this resets the value to its parent value. If the property has not been set in this element then the property is grayed out, to edit the value click on the override button next to the property.

Editing most property types is self explanatory, a few warrant some explanation.
The colour type has four fields which are red, green, blue and alpha. the button brings up a colour picker and is the colour specified by the fields.
Ptr types for things like textures and fonts have an editable text field with the name of the item pointed to and a change button. The change button brings up a list of possible values and '-= new =-' which asks which group file to add the item to, then asks for the location of the file to add. This file is added into the group and the property is set to its name.
Ptr types for things like property sets and materials are like the other pointer types except the '-= new =-' procedure is different. This asks which file to add the item to or should it be added to the current style, then asks for a name for the new item. There is also an extra option '-= inline =-' in the list, this adds a property set (or material) into the element inline.

Element Editor Toolbar

  • Save - Saves the currently open UI file
  • Reload - Abandons changes and reloads the currently open UI file
  • Refresh... - Resends the data on the currently selected item to the Viewer
  • Live Update... - If ticked, this sends all changes made in the Element Editor to the Viewer continuously
